# Phase 0: Task Spec

# Web辩论室方案

## 一、整体架构

基于现有3Agent辩论引擎v3和飞书集成，采用前后端分离架构，新增Web辩论室模块。

### 推荐技术栈
- **前端**: React + D3.js（热力图）+ WebSocket（实时通信）
- **后端**: Node.js（Express）+ Redis（状态管理）+ WebSocket Server
- **数据层**: 飞书API + 知识库缓存

## 二、核心功能实现

### 1. Web辩论室
- 实时辩论界面，展示3Agent辩论过程
- 支持旁观者登录、发言、投票
- 辩论历史回放功能

### 2. 碰撞热力图
- 基于D3.js实现观点碰撞可视化
- 横轴：时间线，纵轴：观点强度
- 颜色深浅表示观点冲突程度
- 实时更新，支持缩放和拖拽

### 3. 旁观者提问
- 旁观者可通过WebSocket实时提问
- 提问经审核后插入辩论流程
- 支持点赞、踩、回复

## 三、实施步骤

### 第一阶段：基础建设（1-2周）
1. 搭建WebSocket服务器，连接现有辩论引擎
2. 实现辩论室基础UI框架
3. 集成飞书用户认证

### 第二阶段：核心功能（2-3周）
1. 实现实时辩论展示组件
2. 开发碰撞热力图算法和可视化
3. 实现旁观者提问功能

### 第三阶段：优化集成（1-2周）
1. 连接2601块知识库，提供辩论参考
2. 性能优化和压力测试
3. 飞书Webhook集成，推送辩论动态

## 四、关键设计

- **状态管理**: Redis维护辩论状态，支持断线重连
- **热力图算法**: 基于观点向量余弦相似度计算碰撞强度
- **提问队列**: 采用优先级队列，保证辩论流畅性
